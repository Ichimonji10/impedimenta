Project 3 Summary

Here's what I think was done really well on this site:

  1. Content. Though I have previously worked with all the languages (and platforms) shown on this site, I have not kept up with them. In the process of creating this site, I had to go back and re-learn the basic syntax of several languages, and even learned a few tricks along the way. If you still doubt me, check out the Android page, or the comments in my Perl source code.
  2. Design, both technical and aesthetic. Yes, I improved the look of the website (I'm no longer using an overly dark tone). But even more than that, I made sure the site was well-constructed. For example, inspect the navbar elements on the left side of the page. In project 2, the nav elements overflowed their container (due to extra border width), while in project 3 they don't (due to inset drop shadows). Tough! There are other small touches all around the site: code blocks add scroll bars when text starts overflowing, the dotted vertical line helps with readability, the logo/graphic will never interfere with main page content (even when the browser window is shrunk down), and pictures are clickable.
  3. Incidental work. In the making of this site, I installed the Android platform and created a "Hello, world!" project from scratch. I installed Apache web server, and figured out how to create a (fairly) secure install. (Really: there are a lot of ways the default Apache install can abused) These projects are outside the scope of the class, but they required a considerable investment of time and effort.

Here's what's not up to par:

  1. Mobile access. I briefly tried making my site mobile-accessible, and then discovered how fragmented support for the mobile landscape really is. I really should persist and figure out how to create a mobile version of the site, but did not.
  2. Server side includes. Each page on the website has several common elements (e.g. navbars), and SSIs would make the site easier to maintain.
