"Hello, world!" in Perl

#!/usr/bin/perl use warnings; use strict; #############################################################################-*\ # # Author(s): # Jeremy Audet # # First Created: 10/24/2011 # License: Creative Commons BY-SA http://creativecommons.org/licenses/by-sa/3.0/ # # Thanks to: # PerlMonks http://www.perlmonks.org/?node_id=111088 # When I last worked with Perl, I enabled warnings by writing a shebang like this: #!/usr/bin/perl -w. The -w switch was necessary for perl before v5.6; v5.6 was released on 03/22/200. (I'm currently running v5.14) The better way to enable warnings is with the more semantically correct "use warnings" and "use strict" commands. # #############################################################################-*/ print "Hello, world!\n";

If your script is more than ten lines long, you should probably have warnings enabled.

The line #!/usr/bin/perl is called a shebang. The shebang tells the computer what type of script is contained within this file (e.g. perl, bash, csh, python). A shebang is necessary because perl is a scripting language, not a compiled language. Scripting languages are interpreted at runtime, rather than being compiled in advance.
